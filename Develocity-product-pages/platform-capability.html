Predictive Test Selection applies a model trained on your Develocity outcome history to score each test by its probability of failing given a specific changeset. Tests below threshold are skipped on PRs and feature branches; full suites run on release builds. Skip decisions are auditable per-run in Build Scan.

Model accuracy

Skip only the tests that won't catch anything

Model trained on your outcome history — not a generic heuristic. Learns the relationship between changed files and failing tests specific to your codebase.

99.1% failure catch rate measured on Spring Boot over a 12-month window of 4,300+ builds. Safety-first by design: uncertain predictions default to run.

Model retrains continuously as new build outcomes arrive. Accuracy improves with usage volume — the more CI runs Develocity ingests, the sharper the predictions.

Every skip decision is scored, timestamped, and surfaced in Build Scan — queryable per test class, per author, per PR.

Branch strategy

PR builds run faster; release builds stay complete

PTS applies only on configured branches — PRs, feature branches, draft builds. Release branches and main always run full suites, unconditionally.

Branch policy is enforced at the Gradle or Maven plugin level — no CI pipeline changes required. Configure once in develocity.testDistribution.predictiveTestSelection.

Composable with Test Distribution: PTS selects the subset to run, TD parallelizes across agents. Combined, the two capabilities compound the time savings.

branch policy config · build.gradle
// develocity.gradle
develocity {
  testSelection {
    predictive {
      enabled = true
      threshold = 0.05
      branches {
        include("feature/*")
        include("pr/*")
        // main, release/* excluded
      }
    }
  }
}
Threshold tuning

Balance speed and safety per project

Default threshold of 0.05 (5% predicted failure probability) is calibrated for maximum safety — typical projects see 40–65% test skip rates at this setting.

Raise threshold for faster builds on well-tested, stable codebases; lower it for projects with high-consequence failures or thin test coverage. Tunable per Gradle subproject.

Develocity 360 surfaces catch-rate vs. skip-rate curves per project — so threshold decisions are data-driven, not guesswork.

Observability

Every skip decision is auditable

Each skipped test records: confidence score, model version, contributing features (changed files, recent outcomes). Queryable in Build Scan and Develocity Analytics.

Aggregate views in Develocity 360: catch rate trend, skip rate by project, false-negative count, and model-version history — so you can see if accuracy drifts.

False-negative tracking: when a skipped test would have failed, Develocity records it. Feeds back into model retraining and is surfaced in the platform dashboard.

Integrations

Works with the build tools you already use

Gradle and Maven supported natively — plugin-based, no agent installation. Requires Gradle 7.1+ or Maven 3.8.1+.

CI-agnostic: runs on GitHub Actions, Jenkins, CircleCI, TeamCity, Buildkite, GitLab CI, and any pipeline that invokes Gradle or Maven builds.

Build Scan telemetry emitted automatically — no additional instrumentation. PTS skip decisions appear in the same Build Scan as the rest of the build data.
